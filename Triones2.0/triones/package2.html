1.       如何提高代码的重用率？

 

提高代码重用率的关键在于理解非结构性重用和结构性重用的区别。

 

非结构性重用主要指不依赖于数据结构的重用模式，它的主要实现方式是Interface。Interface的适用范围是最广泛的，缺少对数据结构的依赖，使得Interface本身非常灵活，但同时也是效率最低的一种重用，所有支持Interface的实体类都必须开发相应的代码，Interface只提供了行为模式上的重用。

 

  非结构性重用的另一种实现方式是与数据结构无关的Abstract Action Class。一般情况下很少有人会设计和实现Abstract Action Class，主要原因在于缺少了数据结构的支持，很难实现与数据直接相关的行为模式，而只能基于已有行为模式的进一步组合，即可用性上的扩展。由于Abstract Action Class 只是作为扩展特征被某个Class继承使用的，而JAVA编程语言本身不提供多重继承的模式，一旦某个类选择继承Abstract Action Class，必然会失去对其他类的继承权，因此很少人愿意选择使用Abstract Action Class。大多数情况下，更倾向于采用Class/Interface Method的方式，提供静态方法，将Class/Interface作为方法参数的一部分，当然这种方式在程序的理解性上显然会大打折扣的。

 

结构性重用主要指提供实现所需的基本数据结构的重用模式，它的主要实现方式是普通意义上的Class。而Abstract Class在某种程度上可以认为是非结构性重用和结构性重用之间的过渡。结构性重用虽然能提供很高的重用率，但由于对数据结构的依赖性，使得其适用范围大大缩小，只能满足个别领域应用目的。

 

采用继承方式提高重用的主要瓶颈在于当某个Class需要实现多个Interface时，最多只能继承一个Interface的实现，而无法利用其它Interface的已有实现，从而降低了代码的重用率。因此，为了有效提高代码重用率，至少要做2件事：一、多实现与结构无关的Interface Method；二、在要实现的Interface的方法中直接调用Interface Method。当然，要从根本上提高重用，最好还是让JAVA支持多重继承。

2.       为方法的参数边界值设计新的重载方法，是否恰当？

 

众所周知，参数是最基本的提供重用的形式，随着方法重载这一概念的出现，对如何高效使用参数提出了挑战。以List接口程序为例：

 

public String add(int index,Object element)//将element插入到指定位置

 

public String add(Object element) //将element添加到最后

 

上面的示例中，我们完全可以通过定义第一个方法的index为0时来替代第2个方法，因此，除了使用上略显方便以外，第2个方法并没有体现出多少实用价值，反而加重了接口本身的维护成本（注释、代码的冗余和更新时要保持一致性的开销），并且也容易造成实现类本身的代码冗余。更糟糕的是，会影响调用者习惯性地延用此类方法的模式，从而造成更大规模的维护成本上升。

 

因此建议，除非为了保护已发布版本的历史延续性，或为了控制单个方法实现的复杂性，或有证据表明边界值的使用频率远高于一般形式，不推荐采用将边界值参数扩展为新的重载方法这样的程序设计风格。

3.       如何应付越来越多的来自使用者的要求？

 

对于一位程序员来说，构件能得到很多人的肯定无疑是件令人兴奋的事。但随着构件本身地不断扩充，原本简单、易懂的特性逐渐丧失，同时复杂性的提高增加了程序员维护和优化构件的难度及测试成本。

 

为了提高程序的可维护性，有必要在保证对外接口的情况下，将方法根据依赖关系逐步分层，简化单个对象程序的复杂度。

 

在编程中可以遵照以下原则进行分层：（注：重用特指可被继承的；通用指可被反复调用的）

 

接口概念（Interface Kernel）：提供最基本的接口实例结果特征，但不确定产生结果特征的实现策略（重用概念）

行为接口（Interface）：在概念接口的基础上确定实现策略，即操纵接口的基本方法（重用控制策略）

标准抽象类（Abstract Class）：提供存储结构并实现接口，但限制（protected）用于构造结构的方法（重用存储结构）

 

抽象类扩展（Abstract Class Extend）：继承标准抽象类的控制策略并扩展策略接口（重用控制策略）

接口方法库（Interface Method）：基于接口扩展的静态方法，作用类似于接口扩展类（重用扩展应用）

活动类（Active Class）：继承于抽象类，提供通用的构造结构的方法（重用构造方法）

 

包含接口类（Include Interface Class）：包含接口，提供基于接口的扩展方法（重用扩展应用）

 

专用扩展类（Special Class）：基于特定类扩展的方法，如相同功能不同变量类型的方法重载等（重用扩展应用）

类方法库（Class Method）：基于特定类扩展的静态方法，作用类似于类扩展（重用扩展应用）

 

采用以上方式不仅改善了代码维护的难度，同时可以在提供多个级别的重用，从而最大限度地发挥构件本身的可重用性。

4.       提高代码的可重用性是否必然会导致性能的降低？如何保证构件的性能？

 

通常评价代码的性能主要集中在响应速度、占用资源、并发性、可靠性和容错性等方面。对于重用构件的设计者来说，面临的主要问题是如何提高构件的响应速度并降低构件占用的资源。
